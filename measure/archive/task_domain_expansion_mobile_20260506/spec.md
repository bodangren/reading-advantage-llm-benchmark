# Track: Task Domain Expansion — Mobile & React Native

## Overview
Expand the benchmark beyond web tasks to include mobile and React Native brownfield integration tasks.

## Goals
- Define React Native task category
- Create sample tasks for native module integration, navigation, and state management
- Update harness to support React Native repo snapshots

## Acceptance Criteria
- [ ] At least 3 React Native tasks in catalog
- [ ] Tasks cover native module bridging, navigation, and async storage
- [ ] Harness can run against React Native repos
- [ ] Scoring rubric accounts for mobile-specific concerns (permissions, platform differences)
- [ ] Tests verify task schema validity

## Non-Goals
- iOS/Android native code tasks (Swift/Objective-C/Kotlin)
- Flutter tasks
